# data/

This folder is the **single source of truth** for all pre-computed, queryable data about the gkniftyheads collection.

All files are plain JSON so they can be served instantly via any static host or CDN (jsDelivr, GitHub Pages, Vercel, Netlify, Cloudflare, etc.).

## Philosophy

- **No database needed** for the public dashboard.
- Everything is pre-calculated nightly so the frontend is blazing fast.
- Files are versioned in Git — full history of how rarity and supply changed over time.
- Split by schema where possible to keep individual file sizes reasonable (30k–50k rows recommended).

## File Inventory

| File / Folder                  | Description                                                                 | Updated By              | Size Expectation (125k) |
|--------------------------------|-----------------------------------------------------------------------------|-------------------------|-------------------------|
| `manifest.json`                | Global sync metadata and collection-level stats                             | hydrate + delta-sync    | Tiny (~1 KB)            |
| `templates.json`               | Master lookup of every template_id → max_supply + immutable traits          | hydrate                 | Small (~100–500 KB)     |
| `schemas/gkniftyheads.json`    | Complete asset records for the main collection (one file per schema)        | hydrate + delta-sync    | ~15–40 MB (uncompressed) |
| `template_stats.json`          | Live per-template statistics (surviving count, burn rate, min/max mint, avg rarity) | calculate-rarity   | Small                   |
| `trait_exposure.json`          | Live percentage of survivors carrying each trait (split into rarity_traits vs variation_traits) | calculate-rarity | Small–Medium            |
| `leaderboard.json`             | Top holders (by weighted rarity sum) + Top 100 rarest surviving assets      | calculate-rarity        | Medium                  |

## Recommended Practices

1. **GZIP compression** (highly recommended)
   - Serve both `.json` and `.json.gz` versions.
   - Modern CDNs and browsers handle this automatically and you save ~70-80% bandwidth.

2. **CDN delivery**
   ```js
   const dataUrl = 'https://cdn.jsdelivr.net/gh/henrikpiga/gkniftyheads-tracker@main/data/leaderboard.json';
   ```

3. **Caching**
   - Set `Cache-Control: public, max-age=3600` (or longer) on these files.
   - The daily workflow updates them once per night.

4. **Schema splitting (future-proofing)**
   - If you add more collections or very large schemas, create additional files under `schemas/`:
     - `schemas/coupons.json`
     - `schemas/profile-heads.json`
   - Update `calculate-rarity.js` and the frontend loader accordingly.

## Data Freshness

- `last_sync_timestamp` in `manifest.json` tells you exactly when the data was last updated.
- All leaderboards and stats are calculated **only on surviving (non-burned) assets**.

## Do NOT edit these files manually

They are generated by the scripts. Manual edits will be overwritten on the next run.

If you need to backfill or correct historical data, re-run `hydrate.js` (or a custom migration script) and then `calculate-rarity.js`.
